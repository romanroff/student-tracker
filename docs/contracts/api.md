# API Contract

The API is versioned under `/api/v1` and implemented by FastAPI.

Authentication accepts Telegram Mini App `initData`, validates signature and
freshness server-side, and establishes the application user. Raw client-provided
Telegram ids never authorize access. Course endpoints require ownership or an
explicit membership role.

Current resources cover session/user, courses, students, lessons, grades, and
attendance. A lesson exposes its user-provided `kind` string and maximum score.
JSON uses stable ids rather than
student or topic names as relationships. Validation errors use a consistent
machine-readable code and field details. Missing and unauthorized resources
must not disclose another tenant's existence.

OpenAPI generated by FastAPI is the executable schema. Integration tests must
cover success, invalid input, expired/forged authentication, role denial,
cross-course ids, conflicts, and transactional failure.
